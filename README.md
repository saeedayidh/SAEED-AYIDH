# موقع سعيد بن عايض الرسمي

موقع شخصي/تعريفي متكامل (عربي/إنجليزي) مع لوحة تحكم كاملة، مبني بـ **Node.js فقط بدون أي مكتبات خارجية** — يعمل فوراً بدون `npm install`.

## المحتوى
- صفحة رئيسية: رسالة ترحيب، تعريف عن سعيد، الإحصائيات، حسابات التواصل، الصفحات، المعرض، الأخبار، المدونة، الأسئلة الشائعة، نموذج اقتراحات وشكاوى.
- صفحة **مركز المساعدة (سعيد سنترهلب)** منفصلة (`/help.html`) مع بحث.
- دعم كامل للغتين العربية والإنجليزية (زر تبديل) ووضع ليلي/نهاري.
- لوحة تحكم (`/admin`) محمية بإيميل وكلمة مرور لإدارة كل المحتوى.

## التشغيل محلياً
يتطلب فقط Node.js (إصدار 18 أو أحدث، بدون تثبيت أي حزم):

```bash
node server.js
```

ثم افتح:
- الموقع: http://localhost:3000
- مركز المساعدة: http://localhost:3000/help.html
- لوحة التحكم: http://localhost:3000/admin

### بيانات تسجيل الدخول الافتراضية للوحة التحكم
- البريد: `admin@example.com`
- كلمة المرور: `ChangeMe123!`

**غيّر هذه البيانات فوراً** من تبويب «الحساب» داخل لوحة التحكم.

## الصور (الشعار والصورة الشخصية)
تم وضع شعار وصورة شخصية مؤقتين (SVG) في `public/assets/logo.svg` و `public/assets/avatar.svg`.
لاستخدام صورك الحقيقية (شعار SBA وصورتك الشخصية)، افتح لوحة التحكم → «الإعدادات العامة» → ارفع الشعار والصورة الشخصية من هناك مباشرة، وستُحفظ وتظهر للجميع تلقائياً.

## إرسال رسائل الاقتراحات/الشكاوى على البريد
الرسائل تُحفظ دائماً داخل لوحة التحكم (تبويب «الرسائل») بغض النظر عن إعداد البريد.
لإرسالها أيضاً كبريد إلكتروني فعلي:
1. أنشئ حساباً مجانياً في https://resend.com
2. فعّل بريدك/نطاقك واحصل على **API Key**.
3. في لوحة التحكم → «الإعدادات العامة»، أدخل:
   - **البريد الإلكتروني لاستقبال الرسائل** (بريدك الشخصي).
   - **مفتاح Resend API**.
   - **البريد المرسل منه** (يجب أن يكون موثّقاً في Resend، أو استخدم `onboarding@resend.dev` للتجربة).
4. احفظ — وستصلك الرسائل الجديدة على بريدك تلقائياً.

> ملاحظة: إرسال البريد يحتاج اتصال إنترنت من السيرفر الذي يشغّل الموقع (يعمل بشكل طبيعي بعد رفع الموقع على أي استضافة).

## رفع الموقع على الإنترنت (مجاناً)
الموقع عبارة عن سيرفر Node.js صغير (بدون قاعدة بيانات خارجية — البيانات محفوظة في ملف `data/data.json`). يمكن رفعه على أي من:

### الخيار 1: Render.com (مجاني وسهل)
1. أنشئ حساباً على https://render.com وربطه بحساب GitHub.
2. ضع هذا المجلد في مستودع (repository) على GitHub.
3. من Render: New → Web Service → اختر المستودع.
4. اضبط:
   - **Build Command:** (اتركه فارغاً)
   - **Start Command:** `node server.js`
5. أضف Persistent Disk (قرص دائم) وامنحه مساراً مثل `/data` لتجنب فقدان البيانات والصور عند كل تحديث (اختياري لكن يُفضّل).
6. اضغط Deploy — سيعطيك رابطاً مباشراً للموقع.

### الخيار 2: Railway.app
1. أنشئ مشروعاً جديداً واربطه بمستودع GitHub لهذا المجلد.
2. Railway سيكتشف Node.js تلقائياً ويشغّل `node server.js`.
3. أضف Volume لمجلد المشروع لحفظ `data/` و `public/uploads/` بشكل دائم.

### الخيار 3: أي VPS (مثل DigitalOcean / Hostinger VPS)
```bash
git clone <رابط المستودع>
cd saeed-website
node server.js
# يفضل استخدام pm2 لتشغيله بشكل دائم:
npm install -g pm2
pm2 start server.js --name saeed-site
```
ثم اربط نطاقك (Domain) عبر Nginx كـ reverse proxy على المنفذ 3000.

## هيكلة المشروع
```
server.js            # السيرفر الرئيسي (Node http، بدون مكتبات خارجية)
lib/store.js         # تخزين البيانات (ملف JSON) + التشفير
lib/sessions.js       # جلسات تسجيل الدخول
lib/multipart.js      # معالجة رفع الملفات
lib/mailer.js         # إرسال البريد عبر Resend
data/data.json        # كل محتوى الموقع (يُنشأ تلقائياً)
public/               # الواجهة (index.html, help.html, admin/, css/, js/, assets/, uploads/)
```

## ملاحظات أمان مهمة
- غيّر بيانات تسجيل الدخول الافتراضية فوراً.
- لا تشارك ملف `data/data.json` لأنه يحتوي على كلمة المرور المشفّرة ومفتاح Resend.
- يُفضّل تشغيل الموقع خلف HTTPS عند رفعه (تتولاه معظم الاستضافات تلقائياً).
